# Collections

Collections reference snippets and provide display modes that let you visualize
the data in different ways.

They also allow you to define fields. For example, you can choose to display a
snippet's title and description â€” each of which is treated as a separate field.

## Display Modes

### Table

Displays the contents in a table view. Each field appears as a column. You can
sort by columns, search within the table, and select multiple items at once

<Figure caption="A collection, shown as a table, with four fields: Done, Title, Description, Priority">
  <img src="/media/docs/collections/table.png" />
</Figure>

### List

Displays the contents as a list. Each entry appears as a separate card, showing
the selected fields. Lists are especially useful for task management scenarios.

<Figure caption="The same collection, shown as a list">
  <img src="/media/docs/collections/list.png" />
</Figure>

### Grid

Displays the contents in a grid layout. It uses the same card concept as the
list view, but arranges the cards in multiple columns.

<Figure caption="The same collection, shown as a grid">
  <img src="/media/docs/collections/grid.png" />
</Figure>

### Timeline

Displays the contents as a chronological timeline. This makes it easy to use a
container as a journal or activity log.

A time range picker lets you focus on specific periods, such as daily, weekly,
monthly, yearly, or all entries.

<Figure caption="The same collection, shown as a timeline">
  <img src="/media/docs/collections/timeline.png" />
</Figure>
